## 1. Authentication Module
`SK_Federations/app/Modules/Authentication/`

### Design System
- **Color theme:** `#213F99` (SK blue) · `#d0242b` (SK red)
- **Gradient:** `linear-gradient(135deg, #213F99 0%, #d0242b 100%)`
- **Font:** Inter
- **Layout:** Split-screen — logo/brand panel on the left, form card on the right
- **Background:** `Background.png` at low opacity with `mix-blend-mode: luminosity` over the gradient
- **Assets:** Served via PHP route from `assets/` folder; published with `php artisan vendor:publish --tag=auth-assets --force`

### Pages

#### `login.blade.php`
- Split layout: left panel shows SK Fed logo + brand name + background image
- Right panel: login card (`width: 516px`, soft layered shadows, `overflow: clip`)
- Heading: "Welcome, SK Federation 👋" with wave emoji animation, `white-space: nowrap`
- Fields: Email + Password
- Sign In button disabled until both fields are filled
- Brand name uses "SK Federation" (singular) in the card heading

#### `verify-wait.blade.php`
- Split layout (same as login)
- Shown after login when email verification is pending
- `form-header h2` set to `1.65rem` to prevent overflow on longer headings

#### `verify-success.blade.php`
- Split layout
- Shown when email verification succeeds

#### `verify-notice.blade.php`
- Split layout
- Informational notice about verification

#### `takeover-wait.blade.php`
- Split layout
- Shown during single-session takeover flow

#### `forgot-password.blade.php`
- Split layout
- Password reset request form

#### `reset-password.blade.php`
- Uses Vite (Kabataan-style) — do NOT change to match auth split layout

#### `password-reset-success.blade.php`
- Shown after a successful password reset
- Split layout (same as login)

## 2. Dashboard Module
`SK_Federations/app/Modules/Dashboard/`

### Assets
- CSS: `assets/css/dashboard.css`
- JS: `assets/js/dashboard.js`
- Published with: `php artisan vendor:publish --tag=dashboard-assets --force`
- Served from: `/modules/dashboard/css/dashboard.css` and `/modules/dashboard/js/dashboard.js`

### Layout Structure
- **Navbar** (fixed, `64px` height) — gradient background
  - Left: hamburger/ellipsis toggle + SK Federations logo + brand name
  - Center: search bar (hidden on mobile ≤768px)
  - Right: notification bell (popover) + profile button (dropdown)
  - On mobile (≤768px): brand name hidden, logo only shown
  - Toggle icon: `fa-bars` when expanded, `fa-ellipsis-v` when collapsed
- **Sidebar** (fixed, `260px` expanded / `72px` collapsed)
  - Top: clickable user avatar + name + role → navigates to Profile
  - Active state on profile page: blue left border + blue name text (`.sidebar-profile-active`)
  - Menu items: Dashboard · SK Community Feed · Barangay Monitoring · Program Monitoring · Kabataan Monitoring · Reports · Logout
  - Collapsed state: icon-only with hover tooltips on desktop
  - Mobile (≤1024px): slides as drawer with overlay
- **Main content:** `margin-left: 260px` (or `72px` collapsed), `margin-top: 64px`

### Navbar — Profile Dropdown
- Items: Profile · Change Password · Logout
- Clicking Profile or the sidebar avatar navigates to the profile page with a loading screen transition

## 3. Profile Module
`SK_Federations/app/Modules/Profile/`

### Assets
- CSS: `assets/css/profile.css` — served via PHP route directly (no publish needed)
- JS: `assets/js/profile.js` — served via PHP route directly

### Layout
- Same navbar and sidebar as Dashboard
- Sidebar profile section gets `.sidebar-profile-active` (blue left border + blue name) when on the profile page

### `profile.blade.php`
- **Profile header:** gradient background (`#213F99` → `#d0242b`) + `Background.png` injected as an `<img>` tag (absolute positioned, `opacity: 0.08`, `mix-blend-mode: luminosity`)
  - CSS `::before` approach was not used because CSS is served via PHP route (relative URLs don't resolve)
- **Content:** Personal Information section only
  - Shows user fields: name, email, role, etc.
  - Edit Profile button opens `edit-profile-modal.blade.php`
- **Removed:** Change Password tab, "My Profile / View and manage..." page header card, Profile menu item from sidebar

### `edit-profile-modal.blade.php`
- Modal overlay for editing personal information
- Triggered by the Edit Profile button on the profile page

### `change-password-form.blade.php`
- Standalone change password form view (kept in module but not linked from the profile page UI)
- Change Password tab was removed from the profile page — users access this via the navbar dropdown → Change Password link

### Navigation to Profile
- Clicking the user avatar/name in the sidebar
- Clicking "Profile" in the navbar profile dropdown
- Both trigger a loading screen transition before navigating

## 4. Key Technical Notes

### Asset Publishing
```bash
# After any auth CSS/image changes:
php artisan vendor:publish --tag=auth-assets --force

# After any dashboard CSS/JS changes:
php artisan vendor:publish --tag=dashboard-assets --force

# Profile assets served directly via route — no publish needed
```

### Hard Refresh
Always do `Ctrl+Shift+R` after publishing assets to bypass browser cache.

### CSS `::before` Limitation
Background images in CSS `::before` pseudo-elements do not work for auth/profile pages because CSS is served via a PHP route — relative URLs in the CSS cannot resolve to public assets. Use inline `<img>` tags in Blade instead.

### Single Session / Heartbeat
Dashboard sends a heartbeat POST every N seconds (configured via `sk_fed_auth.single_session.heartbeat_interval_seconds`) to maintain the active session.

### Back Navigation Lock
All authenticated pages push a history state on load and intercept `popstate` to prevent back-button navigation after logout.
