## SK Federation Profile Page

### Overview
Created a new SK Federation Profile page at `/sk-federation-profile` inside the `CommunityFeed` module. This page serves as the official public-facing profile of the SK Federation Santa Cruz, accessible to logged-in SK officials.

### Route
```
GET /sk-federation-profile → CommunityFeedController@skFedProfile → community_feed::sk-fed-profile
POST /sk-federation-profile/post → CommunityFeedController@createPost → community_feed::sk-fed-profile
```

### Features
- Cover photo using `Background.png` (login background) with blue gradient overlay — "Edit Cover" button removed
- SK Federation logo avatar with edit button
- Stats row: Barangays (26), Officers (12), SK Term (2023–2026)
- Edit Profile button + Create Post button
- Left column: About card (contact, address, office hours), SK Federation Officers list (Chairman, Vice Chairman, Secretary, Treasurer, Auditor, PRO)
- Right column: Compose box with post type buttons (Announcement, Event, Activity, Program) + sample posts with Edit buttons
- Create Post modal — fields: Post Type, Title, Content, Date, Time, Venue
- Edit Profile modal — fields: Federation Name, Contact, Email, Office Address, Description (prototype mode)
- Full SK Federations navbar and sidebar

## Barangay SK Profile Page (SK Federations)

### Overview
Created a dedicated barangay profile page at `/barangay-profile/{slug}` inside the `CommunityFeed` module. SK officials can "stalk" any barangay's SK profile — same design concept as the Kabataan barangay page but using the SK Federations navbar/sidebar and blue theme.

### Route
```
GET /barangay-profile/{slug} → CommunityFeedController@barangayProfile → community_feed::barangay-profile
```

### Supported Slugs
`alipit`, `bagumbayan`, `bubukal`, `duhat`, `gatid`, `labuin`, `pagsawitan`, `san-jose`, `santisima-cruz`

### Features
- Cover using `Background.png` with barangay color gradient + `mix-blend-mode: luminosity`
- Avatar circle with barangay initials, name, location, stats (Posts, Officers, SK Term)
- Left column (sticky on desktop): SK Officers list, SK Councilors grid, Barangay Information, Contact Information
- Right column: Posts feed with tab filter (All / Events / Announcements / Activities)
- Independent scroll — left column sticky, right column scrolls freely with page
- Responsive: single column on ≤1024px

### CSS Custom Property
`--brgy-color` set on `<body>` via inline style — all color references in the stylesheet use `var(--brgy-color)` to avoid Blade expressions inside `<style>` tags.

## Barangay Monitoring — Show Page Rebuild

### Overview
Rebuilt the Monitoring tab of the barangay detail page (`/barangay-monitoring/{slug}`) with a richer program monitoring concept replacing the old KPI/compliance/system data layout.

### New Monitoring Tab Sections

#### 1. Overview KPIs (6 cards)
Icon-only labels on tablet/mobile (text hidden via CSS), full label on desktop:

| Card | Icon | Color |
|---|---|---|
| Total Youth Population | `fa-users` | Blue |
| Programs Created | `fa-tasks` | Purple |
| Ongoing | `fa-spinner` | Amber |
| Completed | `fa-check-circle` | Green |
| Total Participants | `fa-user-check` | Blue |
| Overall Performance Rating | `fa-star` | Orange |

#### 2. Performance Summary
- 3 metric cards: Program Completion Rate, Participant Attendance Rate, Budget Efficiency
- 3 highlight boxes: Top Sector, Most Successful Program, Low Participation count
- Short labels with `text-overflow: ellipsis` and `title` tooltip for full name

#### 3. Programs List Table
Columns: Program Name, Category/Sector, Status (color-coded), Participants, Budget Allocated, Budget Used, Timeline, Reports (PDF links)
- Status filter dropdown (All / Ongoing / Completed / Planned)
- Table headers use icon + text; text hidden on tablet/mobile via `.bm-th-text` CSS class
- Budget, Timeline, Reports columns hidden on tablet; Participants hidden on mobile

### Profile Tab Updates
- Cover now uses `Background.png` with `mix-blend-mode: luminosity` instead of the SK Fed logo watermark
- `bm-header` section removed from both index and show views — back link is now a standalone inline element

### Controller Updates
`barangayDetails()` in `BarangayMonitoringController` extended with:
- `program_stats` — total youth population, programs created, ongoing, completed, participants, overall performance
- `performance_summary` — completion rate, attendance rate, budget efficiency, most active sector, most successful program, low participation count
- `program_list` — flat array of all programs (previous + current + future) with budget and report fields

### Responsive Fixes
- KPI grid: 6 columns → 3 columns at ≤1280px → 3 columns at ≤1024px → 2 columns at ≤480px
- KPI label text hidden on ≤1024px, icon only shown
- Table header text hidden on ≤1024px, icon only shown
- Profile grid stacks to single column on ≤900px

| File | Change |

| `app/Modules/BarangayMonitoring/Views/show.blade.php` | Rebuilt Monitoring tab, updated cover, removed bm-header |
| `app/Modules/BarangayMonitoring/Views/index.blade.php` | Removed bm-header |
| `app/Modules/BarangayMonitoring/Controllers/BarangayMonitoringController.php` | Added `program_stats`, `performance_summary`, `program_list` to `barangayDetails()` |
